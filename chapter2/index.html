What Is Prompt & Context Engineering?
Prompt

A request given to an AI model. It could be:

💬 Question 📋 Instruction 🚫 Constraint ✅ Task
e.g. "Write a Python script to fetch data from a REST API."
e.g. "Why is my code throwing a TypeError?"
Prompt Engineering

The process of designing prompts to get better, more accurate, and more useful outputs from AI.

Context

Additional information that helps AI understand the task better. It could be:

📄 Requirements 🎨 UI/UX Design 💻 Codebase 🐛 Error Logs
e.g. File: api_docs.md (Contains API endpoint schema)
e.g. Error log: TimeoutException at line 42
Context Engineering

The process of providing structured, relevant, and sufficient information to AI systems.

Technique 01 — Persona & System
Technique 1

Persona & System

Set the role, expertise, behaviour, constraint, and tone for the AI before the task begins — for the entire conversation.

Structure
RoleAssign a specific persona and role
ExpertiseDefine the knowledge domain
BehaviorInstruct how to act and respond
ConstraintsSet strict rules on what to avoid
AudienceDefine the target profile for the tone
When to Use

When you need a consistent persona, specific domain expertise, or strict behavioral constraints across a conversation.

References
Role-play prompting consistently surpasses the standard zero-shot approach across most benchmarks. Notably, accuracy on AQuA rises from 53.5% → 63.8% (+10.3%). (Kong et al., 2023; Expert Prompting paper, 2023)
Bad Prompt
Input
"Explain what AI Agents are"
Output
"An AI agent is an autonomous software entity that perceives its environment through sensors and takes actions using effectors to maximize expected utility. It relies on a continuous perception-cognition-action cycle and utilizes complex Markov Decision Processes..."
Generic, jargon-heavy
Good Prompt
Prompt & System
// Role & Expertise
"You are an AI Engineer specializing in Agentic Workflows.

// Behavior & Constraints
Explain technical concepts using simple, everyday analogies.
Rule: Absolutely no academic jargon.

// Tone & Audience
Target Audience: A junior front-end developer transitioning to AI."
Input
"Explain what AI Agents are"
Output
"An AI Agent is an autonomous system that uses an LLM as its core reasoning engine to perceive its environment, use tools, and take actions to achieve a goal.

Think of a standard LLM as a brilliant researcher locked in a library—they can give you the recipe but can't cook. An AI Agent is that same researcher, but with hands, a kitchen, and a credit card.

Instead of just writing code, an Agent executes and tests it. Instead of just planning a trip, it uses APIs to book the flight."
Clear, junior-friendly, with examples
Technique 02 — Few Shot
Technique 2

Few Shot

Provide the AI with context and examples to control the structure and style of the expected output.

Structure
ContextBackground of the situation
ObjectiveThe specific task you want done
RulesConstraints, tech stack, requirements
ExamplesExample the AI should follow
When to Use

When you need the AI to follow a strict format or learn from specific contextual examples not found in general data.

References
Providing in-context examples significantly improves the model's ability to follow strict formatting and complex logic. Few-shot examples pushed GPT-4o pass@1: 47.1% → 57.5% (+10.4%). (CodePromptEval Benchmark, 2024)

Technique 03 — Chain of Thought
Technique 3

Chain of Thought

Guide the AI to solve a problem step by step — showing its reasoning process before giving the final answer.

Structure
ProblemThe core task or question to solve
TriggerExplicit instruction to "Think step by step"
ProcessSequential reasoning steps before result
When to Use

When solving logic puzzles, math, or complex diagnostic tasks where the reasoning process is as important as the result.

References
Eliciting intermediate reasoning steps significantly enhances the model's ability to solve complex arithmetic, commonsense, and symbolic reasoning tasks. CoT prompting shows accuracy improvements of up to +18% on arithmetic tasks. (Wei et al., 2022; Chain-of-Thought Prompting Elicits Reasoning)
Bad Prompt
Input
"Why is the RAG Chatbot response inaccurate?"
Output
"The inaccuracy is almost certainly due to your vector database configuration. You should try swapping your current embedding model for a larger one, as the current one likely lacks the semantic depth needed for your specific domain."
Single guess, no reasoning
Good Prompt
Prompt & System
// Problem Context
"Diagnose the root cause of RAG inaccuracies by evaluating:
Step 1: Vector Embeddings | Step 2: Chunking | Step 3: Retrieval

// Trigger
"Think step by step."

// Process
"Show your reasoning for each component before concluding."
Input
"Why is the RAG Chatbot response inaccurate?"
Output
Step 1: Vector Embeddings
→ Uses text-embedding-3-large (high semantic quality). Pass.
Step 2: Chunking Strategy
→ Documents are chunked at 2,000 tokens. This dilutes context density. Fail.
Step 3: Retrieval Mechanism
→ Top-K is set to 5 (standard). Pass.
Root Cause: Chunk size is too large for the Top-K limit.
Resolution: Reduce chunk size to 512 tokens.
Systematic diagnosis + root fix
Technique 04 — Tree of Thought
Technique 4

Tree of Thought

Guide AI to explore multiple reasoning paths simultaneously, evaluate each one, and select the best solution.

Structure
ProblemDefine the complex task with multiple potential paths
ExploreBrainstorm N different approaches (A, B, C...)
EvaluateCompare trade-offs of each reasoning tree
DecideSelect the optimal path based on logic
When to Use

When there is no single "right" answer, and you need to compare multiple strategic paths or expert opinions.

References
Exploring and evaluating multiple reasoning paths allows models to backtrack and recover from errors during complex problem-solving. ToT achieves 74% accuracy vs GPT-4 default 7.3% on the Game of 24 multi-step task. (Yao et al., 2023; Tree of Thoughts: Deliberate Problem Solving)
Bad Prompt
Input
"Design the best architecture for Identity Verification Agent"
Output
"I recommend a simple monolithic architecture. Use one API endpoint to receive the image, pass it to a multimodal LLM to extract the data and verify the person's identity in one single prompt. This is the most cost-effective way to build it."
One random guess
Good Prompt
Prompt & System
// Problem
"Imagine 3 expert AI architects designing an Identity Verification Agent."

// Explore
Architect A: Single LLM | Architect B: Multi-Agent | Architect C: HITL

// Evaluate & Decide
"Evaluate latency, cost, and accuracy for Fintech. Debate trade-offs, then select the optimal path."
Input
"Design the best architecture for Identity Verification Agent"
Output
// Exploration Phase
Architect A: Lowest latency, highest hallucination risk. Fails compliance.
Architect C: Perfect accuracy, unscalable latency (24+ hours).
Architect B: Moderate latency (2-3s). Agents verify video, OCR, and databases simultaneously. Maximum accuracy.

// Conclusion
[Architect B Selected] Deploying a Multi-Agent swarm architecture.
Evaluates all paths → picks best
Technique 05 — Multi Stages
Technique 5

Multi Stages

Break a complex task into a sequence of smaller, focused prompts — where the output of one stage becomes the input of the next.

Structure
IdentifyDefine core goal and requirements
DesignMap out the system architecture
ExecuteGenerate production-ready code stage by stage
ValidateWrite tests and refine the final integration
When to Use

When a task is too large for a single prompt, involving distinct phases like identification, system design, and validation.

References
Decomposing complex tasks into directed acyclic graphs of smaller prompts drastically reduces hallucination and logical errors. Multi-stage approach vs direct: success rate 29.2% → 78.5% (+49.3%). (Prompt2DAG paper, 2025)
Bad Prompt
Input
"Build a complete AI Price Comparison Agent"
Output
import requests
from bs4 import BeautifulSoup

def get_price(url):
  page = requests.get(url)
  soup = BeautifulSoup(page.content, 'html.parser')
  price = soup.find(id='price').get_text()
  return price

# Note: This is just a basic scraper script.
Incomplete, missing edge cases

// Identify

// Design

// Validate
"Write unit tests for [Stage 3] targeting the edge cases."
